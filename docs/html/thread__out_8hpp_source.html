11/*
12 Possible implementation for multi-threading the digestion of a single
13 sequence. The key thing to note is basically by carefully telling where each
14 digester should start digesting you can make it so each kmer is only
15 considered once. I have very little experience with threading, so you could
16 probably thread this out better than me
17
18 IMPORTANT: This approach will not generate correct results for sequences
19 that contain non-ACTG characters. Take this example, seq = ACTGANACNACTGA, k
20 = 4, l_wind = 4, thread_count = 2, there is a total of 4 valid kmers in this
21 sequence, and thus only 1 valid large window, but we can't know this until it
22 actually goes through the sequence, so it's going to try to partition the
23 sequence into ACTGANACNA, and ANACNACTGA and feed it into 2 digester objects
24 which now each have 0 valid large windows
25*/
